## 版本发布规则

```
V1.82 → V1.83 → V1.84 → V1.85 → V1.86 → V1.87 → V1.88 → V1.89 → V1.90
```

| 规则 | 说明 |
|:----|------|
| 一次只开发一个版本 | 完成→线上测试→发布→再下一个 |
| 每版本改完必须自检 | 5项自检清单（语法/功能/数据库/页面/性能） |
| 改前先备份 | `.bak` + 目录备份 |
| 照葫芦画瓢 | 参考BT5源码对应功能的结构 |
| 敏感不提交 | 带密码/密钥的文件必须先在.gitignore排除 |
| 升级包 | 每次发布附带升级包（仅修改文件+增量SQL） |
